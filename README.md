# 🐛 Bug Tracker
A web-based bug tracking system designed to streamline the process of reporting, assigning, and resolving software bugs. Users can log new bugs with relevant details, assign them to team members, and track the progress of each bug through to resolution.

🔨**Technologies**
HTML,
CSS,
Node.js,
Javascript

**Features**
-**Log a Bug:** Report a new bug with a title, description, and priority level
-**Assign a Bug:** Assign bugs to team members from a dropdown list of existing employees
-**Track Progress:** Monitor the status of each bug from open to resolved
-**Resolve & Close:** Mark bugs as fixed and close them off once resolved

**The Process**
This was a group project built by a team of 4. One member laid the foundation — the HTML structure and CSS colour scheme so the look and feel was consistent before anyone else touched the code. The remaining three handled the JavaScript, making the app dynamic and functional. We divided features amongst ourselves, shared code between the team, and had one person responsible for integrating everything into the final working version.

📚**What I learned**
Working on this project taught me how to collaborate on code as a team and manage combining everyone's work into one codebase. I also learned that instead of using uploaded images for employee profiles, a better approach is to generate avatars. Through this I learned how to create and implement avatars, and how such a small detail can noticeably enhance the look, consistency, and overall user experience of a web application.

**What Can Be Improved**
Rather than allowing users to manually type in employee names when assigning a bug, it is better practice to have employees already loaded into the system and use a dropdown to select from them. This reduces errors and creates a much smoother experience. Other improvements would include connecting a real database so data persists after the page refreshes, adding authentication so only authorised users can assign or resolve bugs, and introducing search and filter functionality so bugs can be sorted by status, priority, or assignee.
